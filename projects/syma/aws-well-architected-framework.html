AWS Well Architected Framework

AWS is no doubt the world's number one cloud and it's sphere of influence is growing each day - customers, partners, developers, architects and the entire eco-system. AWS has published a well architected framework comprising of the consistent set of best practices to be referred to by its customers and partners in order to make sure an architecture is aligned to AWS best practices.

The AWS well architected framework is based on 5 pillars

  • Operational Excellence: This is related to Processes and procedures to deliver business value.
  • Security: This is related to protecting data, applications and systems.
  • Reliability: This is related to the ability of a workload to perform its intended function correctly and consistently when itâ€™s expected to
  • Performance Efficiency: This is related to effectively and efficiently use the cloud resources.
  • Cost Optimisation: This is related to reduce the cost of cloud resources while delivering the business value.

AWS Well Architected Framework uses the following terms

  • Component: A component is the code, configuration, and AWS Resources that together deliver against a requirement.
  • Workload: The term workload is used to identify a set of components that together deliver business value.
  • Architecture: We think about architecture as being how components work together in a workload.
  • Milestone: Milestones mark key changes in your architecture as it evolves throughout the product lifecycle
  • Technology Portfolio: Within an organization the technology portfolio is the collection of workloads that are required for the business to operate.

General Design Principles of AWS Well Architected Framework

  • Stop guessing your capacity needs
  • Test Systems at Production Scale
  • Automate to make architectural experimentation easier
  • Allow for evolutionary architectures
  • Drive architectures using data
  • Improve through game days

Each of the 5 pillars of AWS Well Architected Framework has a set of design principles and the best practices.

Example: Pillar "Security" has the following design principles

  • Implement a strong identity foundation
  • Enable traceability
  • Apply security at all layers
  • Automate security best practices
  • Protect data in transit and at rest
  • Keep people away from data
  • Prepare for security events

And there are following 6 best practices areas for Security

  • Security
  • Identity and Access Management.
  • Detection
  • Infrastructure Protection
  • Data Protection
  • Incident Response
